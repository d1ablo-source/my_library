# 📚 My Library API

Асинхронное REST API для управления библиотекой книг, построенное на современном стеке **FastAPI** и **SQLAlchemy 2.0**. В проекте реализована чистая архитектура с разделением на слои (классический паттерн **Repository**).

---

## 🛠️ Стек технологий
* **Python** 3.10+
* **FastAPI** — высокопроизводительный веб-фреймворк.
* **Pydantic v2** — валидация входных данных и сериализация ответов.
* **SQLAlchemy 2.0** — ORM для безопасного взаимодействия с базой данных (Async).
* **Uvicorn** — быстрый ASGI-сервер.

---

## 🏗️ Архитектура проекта
Проект спроектирован так, чтобы логика роутеров оставалась «тонкой» (только обработка HTTP), а вся техническая работа с БД инкапсулирована в репозитории:

* `main.py` — точка входа в приложение и запуск сервера.
* `database.py` — настройка асинхронного движка базы данных и сессий.
* `models/` — ORM-модели SQLAlchemy (структура таблиц в БД).
* `schemas/` — Pydantic-схемы для валидации запросов и ответов.
* `routers/` — маршруты (эндпоинты), отвечающие строго за HTTP-логику и статус-коды.
* `repository.py` — слой репозиториев (`@classmethod`), содержащий безопасные асинхронные SQL-инструкции (`select`, `update`, `delete`).

---

## 🚀 Реализованные эндпоинты (CRUD)

| Метод | Эндпоинт | Статус | Описание |
| :--- | :--- | :---: | :--- |
| **POST** | `/books` | `201` | Добавление новой книги в библиотеку |
| **GET** | `/books` | `200` | Получение списка всех книг |
| **GET** | `/books/{id}` | `200` | Получение детальной информации о книге по её ID |
| **PUT** | `/books/{id}` | `200` | Полное обновление данных книги |
| **DELETE**| `/books/{id}` | `204` | Удаление книги из базы данных в 1 SQL-запрос |

## 💻 Локальный запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com
   cd my_library
   ```

2. **Разверните и активируйте виртуальное окружение:**
   ```bash
   python -m venv venv
   # Для Windows:
   venv\Scripts\activate
   # Для macOS/Linux:
   source venv/bin/activate
   ```

3. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Запустите сервер разработки:**
   ```bash
   uvicorn main:app --reload
   ```

После запуска интерактивная документация со всеми эндпоинтами будет доступна по адресу:  
🔗 **[http://127.0.0](http://127.0.0)**
